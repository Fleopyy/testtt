Hafta 10 - Nesne Yönelimli Programlama (Soyut Sınıf ve Metotlar)

📚 Soyut Sınıf (Abstract Class) Mantığı

Üzerinden nesne örneği (instance) doğrudan üretilemeyen, sadece başka sınıflara taban oluşturması amacıyla tanımlanan sınıflardır. Ortak özellikleri ve metotları tek bir çatıda birleştirmeyi sağlarlar.

Örnek projede, bankadaki tüm hesapların ortak paydası olan soyut bir Hesap sınıfı tanımlanmıştır. `VadeliHesap` ve `VadesizHesap` bu sınıftan kalıtım alır.

💡 Abstract ve Virtual Metot Farkı

  • Abstract Metot: Gövdesi (kod bloğu) bulunmayan, sadece imzası olan metotlardır. Alt sınıflar bu metotları override etmek ZORUNDADIR.
  • Virtual Metot: Gövdesi olan ve varsayılan bir işi yerine getiren metotlardır. Alt sınıflar isterse aynen kullanır, isterse override edip kendi ihtiyaçlarına göre özelleştirebilir.
Hesap.cs (Soyut Temel Sınıf) 
using System;

namespace Hafta10_24042026
{
    abstract class Hesap 
    {
        private string _hesapNo;
        private string _sahibiAdi;
        private decimal _bakiye;

        public string HesapNo => _hesapNo; // Sadece okunabilir özellik
        public string SahibiAdi => _sahibiAdi;

        public decimal Bakiye
        {
            get { return _bakiye; }
            protected set // Sadece bu sınıf ve türeyen sınıflar değiştirebilir
            {
                if (value < 0)
                    Console.WriteLine("Bakiye 0'dan az olamaz");
                else
                    _bakiye = value;
            }
        }

        public Hesap(string no, string ad, decimal bakiye)
        {
            _hesapNo = no;
            _sahibiAdi = ad;
            Bakiye = bakiye;
        }

        // Sanal Metot: Alt sınıflarda isteğe bağlı ezilebilir
        public virtual void BilgiGoster()
        {
            Console.WriteLine($"{SahibiAdi} - {HesapNo} : {Bakiye}TL");
        }

        // Soyut Metotlar: Alt sınıflar ezmek (override) ZORUNDADIR
        public abstract void ParaYatir(decimal miktar);
        public abstract void ParaCek(decimal miktar);
        public abstract string HesapTuru();
    }
}

📋 Hesap Sınıfı Detaylı Açıklamaları

  • abstract class Hesap: Sınıfın soyut olduğunu belirtir. new Hesap(...) şeklinde doğrudan çağrılarak nesnesi oluşturulamaz.
  • protected set: Bakiye özelliğinin değeri sadece Hesap sınıfı ve ondan türeyen VadeliHesap, VadesizHesap gibi sınıflar içerisinden değiştirilebilir, dış dünyaya kapalıdır.
  • public virtual void BilgiGoster(): Bu metot varsayılan bir çıktı verir. İstenirse alt sınıflar kendi formatında yazdırmak için bu metodu ezebilir.
  • public abstract void ParaYatir(...): Hesap türüne göre para yatırma kuralları değişeceğinden gövdesiz bırakılmıştır. Alt sınıflar kendi kuralları doğrultusunda bu metodu kodlamak zorundadır.
VadeliHesap.cs (Türetilmiş Sınıf) 
namespace Hafta10_24042026
{
    class VadeliHesap : Hesap
    {
        private int _vadeSure;
        private bool _vadeGeldiMi;

        // :base(...) ifadesi üst sınıfın constructor'ını tetikler
        public VadeliHesap(string no, string ad, decimal bakiye, int sure)
            : base(no, ad, bakiye)
        {
            _vadeSure = sure;
            _vadeGeldiMi = false;
        }

        // Soyut metotların ezilmesi
        public override void ParaYatir(decimal miktar)
        {
            if (miktar < 0) return;
            Bakiye += miktar;
        }

        public override void ParaCek(decimal miktar)
        {
            if (!_vadeGeldiMi)
            {
                Console.WriteLine("Hesap vadeli. Süre dolmadan bozamazsınız!");
                return;
            }
            if (miktar > Bakiye) return;
            Bakiye -= miktar;
        }

        public override string HesapTuru() => "Vadeli Hesap";

        // Sınıfa özgü yeni bir metot
        public void VadeSonlandir()
        {
            _vadeGeldiMi = true;
        }

        // Sanal metodun ezilip base metodun çağırılması
        public override void BilgiGoster()
        {
            Console.WriteLine("Hesap Türü: " + HesapTuru());
            base.BilgiGoster(); // Üst sınıftaki BilgiGoster'i çağırır
        }
    }
}

📋 VadeliHesap Sınıfı Detaylı Açıklamaları

  • : base(no, ad, bakiye): Üst sınıf olan Hesap sınıfının kurucusunu çağırır. Alt sınıfta üst sınıfın kurucu parametrelerini doldurmak zorunludur.
  • public override void ParaCek: Vadeli hesaptan para çekebilmek için _vadeGeldiMi kontrolü yapılmıştır. Eğer vade dolmamışsa işlem engellenir.
  • base.BilgiGoster(): Override ettiğimiz metot içerisinde üst sınıfta yazılı olan kodları aynen çalıştırmak için kullanılır. Böylece ortak bilgileri tekrar yazmadan ekrana basabiliriz.

Hafta 12 - Arayüzler (Interfaces)

📚 Arayüz (Interface) Nedir?

Sınıflara belirli yetenekler kazandıran, içerisinde sadece metot ve özelliklerin imzasını barındıran şablonlardır. Sınıfların dış dünyaya sunacağı sözleşmelerdir.

C# dilinde çoklu kalıtım (multiple inheritance) yasaktır. Yani bir sınıf sadece bir sınıftan kalıtım alabilir. Ancak, bir sınıf birden fazla arayüzü uygulayarak çoklu yetenek kazanabilir.

💡 Arayüz Kuralları

  • Arayüzlerin içerisindeki üyeler varsayılan olarak public erişim belirleyicisine sahiptir ve gövdeleri (kodları) yazılmaz.
  • Arayüzlerin kurucu metodu (constructor) ve üye değişkenleri (field) olamaz.
  • Arayüzü uygulayan bir sınıf, arayüzdeki tüm metotları yazmak zorundadır. Yazarken override kelimesi kullanılmaz.
Program.cs (Arayüz Tanımlamaları ve Uygulama) 
using System;

namespace Hafta12_08052026
{
    // Arayüz 1
    interface IGonderilebilir
    {
        void Gonder(string alici, string adres);
        string TakipNoUret();
    }

    // Arayüz 2
    interface ITakipEdilebilir
    {
        string TakipSorgu(string takipNo);
        bool TeslimEdildiMi(string takipNo);
    }

    // Arayüz 3
    interface IFiyatlandirilabilir
    {
        decimal Hesapla(double kg, double mesafe);
        void FaturaGoster();
    }

    // Soyut Sınıf arayüzleri uygular (Çoklu arayüz uygulaması)
    abstract class Kargo : IGonderilebilir, ITakipEdilebilir, IFiyatlandirilabilir
    {
        private string _gondericiAdi;
        private string _takipNo;
        private double _agirlikKg;
        private bool _teslimEdildi;

        public string GondericiAdi => _gondericiAdi;
        public string TakipNo => _takipNo;
        public double AgirlikKg => _agirlikKg;
        public bool TeslimEdildi => _teslimEdildi;

        public Kargo(string ad, double kg)
        {
            _gondericiAdi = ad;
            _agirlikKg = kg;
            _takipNo = string.Empty;
            _teslimEdildi = false;
        }

        // Soyut metot olarak bırakılan arayüz metotları. Alt sınıflar override edecek.
        public abstract void Gonder(string alici, string adres);
        public abstract string TakipNoUret();
        public abstract decimal Hesapla(double kg, double mesafe);
        public abstract void FaturaGoster();

        // Arayüz metotlarının gövdeli virtual olarak uygulanması
        public virtual string TakipSorgu(string takipno)
        {
            if (_takipNo == takipno)
            {
                return $"Takip no {takipno}: " + (TeslimEdildi ? "Teslim Edildi" : "Yolda");
            }
            return "Kargo bulunamadı";
        }

        public virtual bool TeslimEdildiMi(string takipno) => _teslimEdildi;

        protected void TakipNoAta(string no) => _takipNo = no;
        public void TeslimAl() => _teslimEdildi = true;
    }

    // Türetilmiş Sınıf
    class KaraKargo : Kargo
    {
        private string _plaka;

        public KaraKargo(string ad, double kg, string plaka) : base(ad, kg)
        {
            _plaka = plaka;
        }

        public override void Gonder(string alici, string adres)
        {
            string no = TakipNoUret();
            TakipNoAta(no);
            Console.WriteLine($"Kara kargosu {no} takip no ile yola çıktı.");
        }

        public override string TakipNoUret() => DateTime.Now.Ticks.ToString();
        public override decimal Hesapla(double kg, double mesafe) => (decimal)(mesafe * 2.5 + kg * 4.0);
        public override void FaturaGoster() => Console.WriteLine($"Gönderici: {GondericiAdi}, Plaka: {_plaka}");
    }
}

📋 Kargo ve KaraKargo Analizi

  • interface IGonderilebilir: Arayüzler interface anahtar kelimesi ile deklare edilir. Sadece metodun dönüş tipi, adı ve parametrelerini barındırır.
  • Kargo : IGonderilebilir, ITakipEdilebilir, IFiyatlandirilabilir: C#'ta çoklu yetenek kazandırma. Kargo sınıfı 3 ayrı arayüzün de sözleşmesini imzalamıştır.
  • public abstract void Gonder(...): Kargo sınıfı abstract bir sınıf olduğu için arayüzden gelen metotları kendisi gövdeli yazmak zorunda değildir. Bunları alt sınıflara devretmek için abstract olarak tanımlamıştır.
  • class KaraKargo : Kargo: KaraKargo normal (somut) bir sınıf olduğu için Kargo sınıfındaki tüm abstract üyeleri override etmek zorundadır.

Hafta 13 - Entity Framework Core (EF Core) & CRUD İşlemleri

📚 Entity Framework Core Nedir?

C# nesnelerimiz ile ilişkisel veritabanımız (SQL Server vb.) arasında bir köprü vazifesi gören Nesne-İlişkisel Eşleme (ORM) kütüphanesidir. SQL komutları yazmadan verileri C# metotları ile yönetmemizi sağlar.

DbContext: EF Core'un kalbidir. Veritabanı bağlantısını, konfigürasyonunu ve sorgulamaları yöneten ana sınıftır.

DbSet<T>: Veritabanında bir tabloyu temsil eden sınıftır. CRUD işlemleri bu özellik üzerinden yürütülür.

💡 CRUD İşlem Metotları

  • Create (Ekleme): DbSet.Add(entity) metodu ile veri kuyruğa eklenir.
  • Read (Sorgulama): LINQ metotları (Where, OrderBy, ToList, FirstOrDefault) ile veriler çekilir.
  • Update (Güncelleme): Veritabanından çekilen nesnenin özellikleri değiştirilir. EF Core değişikliği otomatik izler (tracking).
  • Delete (Silme): DbSet.Remove(entity) metodu ile veri kuyruktan silinir.
  • Fiziksel Kaydetme: Yapılan işlemlerin veritabanına aktarılması için mutlaka SaveChanges() metodu çağrılmalıdır.
OkulContext.cs & Program.cs 
using System;
using System.Linq;
using Microsoft.EntityFrameworkCore;

namespace Hafta13_15052026
{
    // Entity (Tablo Modeli)
    public class Ogrenci
    {
        public int Id { get; set; } // Primary Key (Otomatik Artan)
        public string AdSoyad { get; set; }
        public int Sinif { get; set; }
        public string Bolum { get; set; }
    }

    // Veritabanı Context Sınıfı
    class OkulContext : DbContext
    {
        // Tabloyu temsil eder
        public DbSet<Ogrenci> Ogrenciler { get; set; }

        // Bağlantı yapılandırması
        protected override void OnConfiguring(DbContextOptionsBuilder optionsBuilder)
        {
            optionsBuilder.UseSqlServer("Server=SQLEXPRESS;Database=OgrenciDB;Trusted_Connection=true;TrustServerCertificate=true;");
        }
    }

    class Program
    {
        static void Main(string[] args)
        {
            using (OkulContext db = new OkulContext())
            {
                // Veritabanı yoksa oluşturulmasını sağlar
                db.Database.EnsureCreated();

                // 1. CREATE (Veri Ekleme)
                Ogrenci yeniOgr = new Ogrenci { AdSoyad = "Sevim Güler", Sinif = 2, Bolum = "Bilgisayar" };
                db.Ogrenciler.Add(yeniOgr);
                db.SaveChanges(); // DB'ye fiziksel olarak kaydeder

                // 2. READ (Veri Çekme & Filtreleme)
                var bilgisayarcilar = db.Ogrenciler
                    .Where(o => o.Bolum == "Bilgisayar")
                    .OrderBy(o => o.AdSoyad)
                    .ToList();

                // 3. UPDATE (Veri Güncelleme)
                var guncellenecek = db.Ogrenciler.Where(x => x.AdSoyad == "Sevim Güler").FirstOrDefault();
                if (guncellenecek != null)
                {
                    guncellenecek.Sinif = 3;
                    db.SaveChanges();
                }

                // 4. DELETE (Veri Silme)
                var silinecek = db.Ogrenciler.Where(x => x.Id == yeniOgr.Id).FirstOrDefault();
                if (silinecek != null)
                {
                    db.Ogrenciler.Remove(silinecek);
                    db.SaveChanges();
                }
            }
        }
    }
}

📋 EF Core CRUD Analizi

  • db.Database.EnsureCreated(): Local veritabanı sunucusunda veritabanının fiziksel olarak var olup olmadığını kontrol eder, yoksa C# modellerine göre otomatik oluşturur.
  • db.Ogrenciler.Add(...): Eklenen nesneyi EF Core Change Tracker mekanizmasında "Added" (Eklendi) durumuna getirir. Henüz SQL sunucusuna gönderilmemiştir.
  • db.SaveChanges(): Değişiklik izleyici (Change Tracker) üzerindeki tüm bekleyen işlemleri (Ekleme, Güncelleme, Silme) analiz eder, uygun SQL sorgularını üretir ve tek bir işlemde veritabanında çalıştırır.
  • FirstOrDefault(): Sorguya uyan ilk elemanı getirir, eğer eşleşen kayıt bulunamazsa C#'taki varsayılan nesne karşılığı olan null değerini döndürür.

Soru 1 - C#'ta Soyut Sınıf Tanımı OOP

C# dilinde doğrudan nesne örneği üretilemeyen ve sadece kalıtım tabanı oluşturabilen sınıflara sınıf denir. Bu sınıflar içinde gövdesi yazılmayan metotlar anahtar kelimesi ile bildirilirken, alt sınıflarda ezilebilecek gövdeli metotlara ise metot adı verilir.
İpucu: abstract / abstract / virtual (Büyük/küçük harfe dikkat edin!)

Soru 2 - Çoklu Yetenek Kazandırma Arayüz

C#'ta sınıflar arasında çoklu kalıtım yasak olduğundan, bir sınıfa birden fazla davranış veya yetenek kazandırmak amacıyla kullanılır. Sınıfın bu yapıdaki tüm metot şablonlarını gövdeli hale getirmesi durumdadır.
İpucu: interface / zorunlu

Soru 3 - Ata Sınıf Kurucu Tetikleme OOP

Türetilmiş bir sınıfın kurucu (constructor) metodundan üst sınıfın kurucu metoduna parametre geçirmek amacıyla kurucu metod imzasının hemen sonuna anahtar kelimesi zincirlenir (Örn: : base(param1)).
İpucu: base

Soru 4 - Verileri Veritabanına Yansıtma EF Core

Entity Framework Core kütüphanesinde kuyrukta bekleyen tüm veri tabanı işlemlerini (Ekleme, Silme, Güncelleme) fiziksel sunucuya yansıtmak amacıyla metodu çağrılmalıdır. LINQ sorgusunda kritere uyan tek bir kaydı almak için ise metodu kullanılır.
İpucu: SaveChanges / FirstOrDefault (C# PascalCase yazım standartlarına dikkat!)

Soru 5 - EF Core Temel Yapı Taşı EF Core

Veritabanı bağlantımızı yöneten ana EF Core sınıfı iken, bu bağlantı içerisinde C# nesnemiz ile veritabanı tablosu eşleşmesini sağlayan özellik türü ise veri tipidir.
İpucu: DbContext / DbSet
